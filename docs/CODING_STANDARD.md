# 13. CODING STANDARDS

## Backend Standards

* Business logic forbidden inside controllers.
* Validation must use FormRequest.
* Use service layer.
* Use enums for statuses.
* Use repository pattern when necessary.
* Use type declarations.

---

## Frontend Standards

* Reusable components.
* Separate layout components.
* Avoid duplicated UI logic.
* Use modular structure.

---

## Git Standards

* Small commits.
* Meaningful commit messages.
* Feature branch workflow.

Commit Examples:

* feat: implement booking validation
* fix: resolve payment calculation issue
* refactor: simplify arena service logic
